Tools exercises:

Please see the general notes at the bottom of this page.

There is no particular time expected for these. You should do them in order, but it's OK to spend more time on the ones that interest you and go quickly through others. Generally, students get through the Test and Performance Issues groups in Monday's exercise period, and may start the Memory Issues exercise. They then start Tuesday's exercise period with finishing up anything they want to from Monday, complete Memory Issues and Code Management and do as much of Build and Release Management as they find useful.

Nobody does every bit of every exercise, and that's OK. Quite a few groups will probably stop around the end of the CMake exercises.

If you run out of time and want to do more on particular topics, you can come back to these topics later in the week or load a copy onto your own machine.

The very first step is get the School's virtual machine installed and running on your laptop, and then to log in. For more information on how to do that, see the School's Laptop configuration page.

If you haven't done this yet, open a Terminal window and get a copy of the exercise files: 

cd ~
git clone https://github.com/bobjacobsen/csc-exercises.git
Finally, so a bit of setup: 
source csc-exercises/CSC/setup

Now move on the the instructions for each exercise.

General notes:

These exercises involve some minimal Java and/or C++ programming. We recommend that you arrange yourself in groups so that somebody in each group has at least a little experience with Java and/or C++.

Note that the instructions for these exercises are written for a Bash-type shell. If you use a C-type shell:

  1. Replace "export SOMETHING=..." with "setenv SOMETHING ..."
  2. Replace "source setup.sh" with "source setup.csh"

Style convention: Indented commands in code font are meant to be cut and pasted into a command line: 

    cd
    echo "Done!"
When we want to show the output from a command, we'll include the command prompt: 
$date
Thu June 15 10:37:13 PDT 2015
That's not meant to be cut-and-pasted in its entirety, just the parts after the $ or % command prompt.
